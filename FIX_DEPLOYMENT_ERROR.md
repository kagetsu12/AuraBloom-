# 修复 "Invalid request body" 错误

## 问题
在更新 Cloudflare Pages 构建配置时出现 "Invalid request body" 错误。

## 可能的原因
1. 部署命令格式不正确
2. 项目名称不匹配
3. 命令中包含了特殊字符或格式问题

## 解决方案

### 方案 1：使用最简单的部署命令

如果部署命令不能留空，尝试使用：

```bash
echo "Deploying"
```

这是一个最简单的命令，不会执行任何实际操作，但可以满足"不能留空"的要求。

### 方案 2：检查项目名称

1. 在 Cloudflare Pages 中查看你的项目名称
2. 如果项目名称不是 "aurabloom"，需要修改命令中的 `--project-name` 参数

### 方案 3：分步配置

1. **先只修改构建命令**
   - 构建命令：留空
   - 部署命令：暂时保持原样
   - 保存

2. **然后修改部署命令**
   - 部署命令：`npx wrangler pages deploy .`
   - 保存

### 方案 4：使用正确的 Pages 命令格式

确保使用正确的命令格式：

```bash
npx wrangler pages deploy . --project-name=你的实际项目名
```

**注意：**
- 项目名应该是你在 Cloudflare Pages 中创建项目时使用的名称
- 可以在 Pages 项目列表页面查看项目名称

### 方案 5：如果以上都不行

尝试使用这个最小化命令：

```bash
true
```

这个命令总是返回成功，不会执行任何操作。

## 推荐配置

### 如果必须填写部署命令：

```
构建命令: (留空)
部署命令: npx wrangler pages deploy .
非生产分支部署命令: (留空)
路径: /
```

### 如果仍然报错：

```
构建命令: (留空)
部署命令: echo "Deploying static site"
非生产分支部署命令: (留空)
路径: /
```

## 检查步骤

1. 确认项目名称是否正确
2. 尝试使用最简单的命令（如 `echo "Deploy"`）
3. 如果还是报错，检查是否有其他必填字段未填写
4. 尝试刷新页面后重新配置
